# Distraction: осознанный досуг = `break`, не `distracted`

## Overview

`distractionPullback` дёрнул юзера в 22:04 на 25-минутной сессии музыки на
YouTube (5 клипов подряд: Alai Oli → Sona Jobarteh → Noize MC → Atlantida →
Константин Ступин). Судья выдал `distracted` conf 85 → пинг.

Это ложное срабатывание. Музыка вечером — не залипание, а отдых. Корень — в
системном промпте судьи ([distractionPullback.judge.ts:32](../../../packages/server/src/cognition/handlers/distractionPullback.judge.ts#L32)):
он держит «ленту YouTube» эталоном отвлечения и **не различает выбранный досуг
(музыка, фильм, игра) от бесконечной ленты**. Музыкальные клипы выглядят как
досуговая лента → высокая уверенность в `distracted`.

**Цель:** научить судью считать `distracted` **только** бесконечные/ленточные
форматы, а осознанно выбранный досуг — `break`, в любой час.

Граница выбрана юзером: **по типу контента** (не по времени суток, не по «дрейфу
с работы»).

## Scope

**In:**

- Переписать `SYSTEM_PROMPT` и описания enum `verdict` в
  `distractionPullback.judge.ts` — зашить границу feed-vs-выбранный-досуг.

**Out (осознанно не трогаем):**

  смене заголовка (§2.6) оставляем (см. Decisions).
- `distractionPullback.ts` handler — без изменений. `shouldPing` уже завязан
  только на `verdict === 'distracted'`, так что `break` сам глушит нудж.
- Схема `JudgeResult` / `report_verdict` tool, конфиг (`envInt`), Discord-эмбеды,
  кнопки — без изменений.
- Тихие часы / время суток как гейт — отклонено юзером в пользу контентной
  границы.
- Историческая строка `distraction_evals` id=39 (музыка, помечена `distracted`)

## Current state

- Судья ([distractionPullback.judge.ts:32-42](../../../packages/server/src/cognition/handlers/distractionPullback.judge.ts#L32)):
  4 вердикта (`distracted` / `break` / `working` / `unknown`). Различает работу
  и досуг по заголовкам, но `break` определён размыто («законный, осмысленный
  перерыв»), а единственный конкретный пример отвлечения — «лента YouTube». Нет
  понятий «музыка/выбранный досуг» vs «бесконечная лента».
- `shouldPing = verdict === 'distracted' && confidence >= confidencePct`
  ([distractionPullback.ts:122](../../../packages/server/src/cognition/handlers/distractionPullback.ts#L122)).
  `break`/`working`/`unknown` уже → skip, запись eval без пинга.

## Design

### 1. Новая граница в промпте

`distracted` (дёргаем) — **только** бесконечные/ленточные форматы. Признак:
единица потребления — лента/поток, а не выбранная вещь.
- short-form ленты: YouTube Shorts, Reels, TikTok;
- ленты соцсетей: X/Twitter, Instagram, Facebook, Reddit;
- новостной скролл, бесконечная лента коротких рекомендаций.

`break` (молчим, любой час) — **осознанно выбранный досуг**. Признак: выбрана
конкретная вещь, у неё есть начало и конец.
- музыка: трек / альбом / концерт / плейлист — **даже перескакивая между ними**;
- фильм / сериал / длинное видео; игра; лонгрид / статья.

`working` — это работа (без изменений). `unknown` — заголовки пустые/неинформа­
тивные (без изменений).

Сохранить существующий приоритет точности: «сомневаешься — НЕ помечай
`distracted`».

### 2. Что меняется в коде

- `SYSTEM_PROMPT` — переписать определения `distracted` и `break` под §1, добавить
  явные примеры (музыка/концерт/фильм/игра = break; shorts/reels/лента соцсетей/
  новостной скролл = distracted) и ключевой различитель «выбранная вещь vs
  бесконечная лента».
- Описание `verdict` enum в `VERDICT_TOOL.input_schema` — синхронизировать с новой
  формулировкой (сейчас: «distracted — дрейф в отвлечение; break — законный
  перерыв»).

`buildJudgePrompt`, парсинг tool_use, `JudgeResult` — без изменений.

## Decisions / defaults

- **Граница по контенту, не по времени.** Тихие часы (как у email/weather) и
  «дрейф только с работы» рассмотрены и отклонены юзером.
- **Ре-эвал на смене заголовка (§2.6) оставляем.** Музыка теперь `break` → детектор
  будит судью на каждом новом треке. Глушить ре-эвал после `break` опасно: тогда
  ослепнем к реальному flip'у внутри того же app-run (музыка → открыл ленту X в
  том же Chrome). Стоимость ограничена `dailyCap=40` + дешёвый Haiku — принимаем.
- **Handler не трогаем** — `break` уже глушит пинг через `shouldPing`.

   - музыка 2026-06-05 21:39→22:04 (5 клипов из `window_history`) → **не**
     `distracted` (ожидаем `break`);
   - реальный кейс ленты-скролла (shorts / лента соцсети) → `distracted`.

   Прогон через настоящий Haiku — ручной/скриптовый, фиксируем результат как
   доказательство.
